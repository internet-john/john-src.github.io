CSS Anchors

A simple way to make your CSS far easier to reason about and debug is through the use of anchor styles. To do so, simply apply defaults to the html and body tags. Today, we will focus on a font-size anchor

Anchors away!

        
        html { 
            font-size: 100%; 
        }

Nearly every browser will have a default font size of 16px.

What follows

Now that we have a default font-size and known body dimensions, we can more easily make use of dynamic sizing for our application content.

REM - "root em"

REM refers to the root element’s font size. Regardless of the nesting of the element you wish to style, where it’s located, etc, anything styled with the REM unit will be in direct calculation of the root element(html).

Above, we set the font-size to 100% (16px). Therefore, we can safely reason about font sizing when using REM. 

         
        btn--cta {
            font-size: 1.5rem;
        }
            
        __

        1.5rem = (16*1.5) = 24px.

TL;DR

Use CSS anchors to have explicit values to refer to better understand size calculations.

Always, always use dynamic sizing units (%, VH, VW, em, rem).

Generally, % and VH/VW are used in conjunction. As for EM and REM, once you use one of them, try and stick to it and not mix. REM is my choice, as for the reasons stated above.
